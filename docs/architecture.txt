2. CORE DATA FLOW
================================================================================

  Python Script
       |
       v
  SneppX_ALG Python Module  (pure-Python fallback if C ext unavailable)
       |
       v
  _SNEPPX_c C extension  (pybind11 bindings)
       |
       v
  SNEPPXTensor / SNEPPXVariable / SNEPPXModel  (C objects)
       |
       v
  SIMD GEMM (AVX2/AVX-512) or CUDA Kernels (GPU)
       |
       v
  Result returned to Python via NumPy interop

================================================================================
3. MEMORY MODEL
================================================================================

  - Polymorphic allocator with global pool + per-thread TLS cache
  - SNEPPX_secure_malloc / SNEPPX_secure_free for sensitive data
  - Guard pages around security-critical allocations
  - CUDA: memory pool, stream pool, event pool, pinned/managed memory
  - Configurable pool sizing and cache limits

4. DISTRIBUTED ARCHITECTURE
================================================================================

  +------------------+  +------------------+  +------------------+
  |  Rank 0 (Node)   |  |  Rank 1 (Node)   |  |  Rank N (Node)   |
  |  +------------+  |  |  +------------+  |  |  +------------+  |
  |  | ZeRO-3     |  |  |  | ZeRO-3     |  |  |  | ZeRO-3     |  |
  |  | Shard      |  |  |  | Shard      |  |  |  | Shard      |  |
  |  +------------+  |  |  +------------+  |  |  +------------+  |
  |  DDP | PP | TP  |  |  DDP | PP | TP  |  |  DDP | PP | TP  |
  +------------------+  +------------------+  +------------------+
         |                      |                      |
         +------ NCCL / RDMA / gRPC / Socket ---------+
                        |
                   All-Reduce / All-Gather / Reduce-Scatter

  - Data Parallelism (DDP): Bucket-based gradient all-reduce
  - Pipeline Parallelism (PP): 1F1B schedule with microbatches
  - Tensor Parallelism (TP): Row/column split linear layers
  - Expert Parallelism (EP): Expert-to-rank mapping (MoE)
  - ZeRO-1/2/3: Optimizer state/gradient/parameter partitioning
  - Elastic Training: Node join/leave/failure with checkpoint recovery
  - Heartbeat: UDP-based health monitoring with SUSPECT->DEAD state machine

================================================================================

  S0 - Build Integrity ---------> SBOM, Reproducible Builds, TUF
  S1 - Memory Hardining --------> Guard Pages, Canaries, ASLR, W^X
  S2 - Code Obfuscation --------> CFG Flattening, VM Obfuscation
  S3 - Runtime Monitoring ------> Integrity, Container Breakout, ML Anomaly
  S4 - Network Security --------> TLS 1.3, Noise, DDoS, 4-Ring Firewall
  S5 - AI Safety ---------------> RLHF, DP-SGD, Prompt/Output Filters
  S6 - AI Sanitizer ------------> Semantic Injection, Encoded Attack Decoder
  S7 - Supply Chain ------------> TUF Signed Updates, A/B Partitions, TPM
  S8 - Formal Verification -----> TLA+ Parser, LTL Model Checking
  S9 - Penetration Testing -----> CVE Scanner, Fuzzer, Compliance Checks

================================================================================
6. BUILD SYSTEM
================================================================================

  - CMake 3.16+ with MSBuild (Windows) or Make/Ninja (Linux/Mac)
  - GLOB_RECURSE auto-discovers .c/.cpp/.asm files
  - Conditional CUDA support via -DSNEPPX_BUILD_CUDA=ON
  - Conditional Python bindings via -DSNEPPX_BUILD_PYTHON=ON
  - Sanitizer builds via scripts/run_sanitizers.sh / run_sanitizers.ps1
  - Multiple build directories for different configurations
